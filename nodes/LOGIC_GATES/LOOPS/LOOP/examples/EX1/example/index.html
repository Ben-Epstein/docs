example

In this example, we begin with a CONSTANT node that has a value of 8.

The value is then passed to a LOOP node, which, in turn, passes it to the nodes connected to its body output. The LOOP node is a special node that takes a parameter called num_loops, indicating the number of iterations. In this example, we used 5 iterations, but the default value for this parameter is -1, meaning it will iterate infinitely. Each time the LOOP node is executed, it checks the remaining number of iterations and enqueues all nodes connected to its body output if there are more iterations to be performed.

Next, we have two nodes, CONSTANT and FEEDBACK, connected to the body output of the LOOP node. The FEEDBACK node is another special node designed to collect the result of a given node ID. It takes a parameter called node_reference, which is the ID of a node. It attempts to retrieve the result of the specified node ID. If the result is not found, it passes the result of its parent node. In this example, we provided the ID of the MULTIPLY node as the parameter for the FEEDBACK node.

We also have a MULTIPLY node, which performs element-wise multiplication on two input Numpy arrays. It is connected to a visual node called BIG_NUMBER, which visualizes the returned value of the MULTIPLY node in an indicators plot.

To enable the LOOP node to function correctly, another helper node called GOTO is required. This node takes a parameter called node_reference, similar to the FEEDBACK node. However, the GOTO node always enqueues the specified node, which, in this case, is the LOOP node. Therefore, the workflow is as follows: the LOOP node enqueues the nodes connected to its body output, then the GOTO node enqueues the LOOP node again. The LOOP node checks if the remaining number of iterations is greater than zero. If so, it enqueues its body nodes again until the number of remaining iterations becomes zero. Finally, it enqueues the nodes connected to its end output.

Lastly, we have an END node which is connected with end output of LOOP node, which serves to terminate the program.
